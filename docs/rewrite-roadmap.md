# Rewrite Roadmap

Arcana is being rewritten under a hard language freeze until selfhost.
This roadmap tracks the current rewrite slice, not the full post-selfhost program.

Imported-source guardrails:
- archived MeadowLang app/toolchain corpus was a bootstrap hotspot import and must not be reintroduced as an implementation template
- `std` itself is rewrite-owned first-party library surface; historical imports only provided transition context
- future Arcana-owned app/media grimoire responsibilities are frozen by role, not by carried Meadow-era package names
- rewrite-owned app/media grimoires now live under `grimoires/owned/*`; the broad historical reference tree is archived outside the repo
- rewrite-owned app/media grimoires under `grimoires/owned/*` now compile on the new frontend and are anchored in crate-side regression checks rather than only ad hoc manual verification
- imported standard-library contents do not automatically define correct standard-library layering; showcase/game/demo logic that leaked into `std` should move back into showcase or app grimoires unless it is explicitly ratified as general-purpose std surface
- first-party host/io/window/input/canvas/time/audio plus primitive graphics/text remain required before selfhost so the rewrite can show real apps/showcases and bootstrap future Arcana-owned grimoires, but the old MeadowLang VM/bytecode/window stack is not the mandated implementation path
- those host/app/runtime packages are real Rust rewrite commitments, not a promise that imported `std` or carried grimoires will keep satisfying them unchanged
- third-party Rust crates may still appear underneath the runtime/backend, but only as replaceable private implementation details rather than the defining first-party substrate
- parser opaque-expression fallback and runtime/resource handles are different concerns: the former should be eliminated from the frontend path before selfhost, while current source-declared opaque std handles such as `Window`, `Image`, `FileStream`, `AppFrame`, and `Audio*` remain bootstrap seams and must be revisited once the rewrite-owned runtime/backend resource model is settled
- ECS scheduling/components remain first-party language/runtime surface during the rewrite; they are not merely showcase helpers carried from MeadowLang
- carried `std.app` fixed-step helpers and `std.tooling` planner helpers are convenience corpus, not rewrite-owned architecture until a scope explicitly ratifies them
- carried generated snapshot artifacts such as direct-emit spec shards are not rewrite architecture evidence by themselves; use them to track migration work, not to override approved contracts or crate-side implementation status
- historical MeadowLang reference corpus is now archived outside the repo and no longer participates in normal default validation.

## Next Milestones

1. Completed: replace the remaining raw opaque-expression fallbacks and leftover grouped-comma bracket ambiguities with fully structured parsing on the current frontend path.
2. Completed: freeze the rewrite-owned package boundaries and bootstrap ledgers: host-core in `selfhost-host/v1-scope`, app/runtime substrate in `app-substrate-v1-scope`, std scope/status, grimoire role scope/status, ECS/behaviors kept first-party, and carried convenience layers explicitly left unratified.
3. Completed: extend the typed frontend from declaration-surface plus body-resolution checks into conservative expression typing, ownership, and borrow/lifetime flow on the current frontend path.
4. Completed: move package fingerprints from declaration-surface/source hashes to typed-HIR/API fingerprints.
5. Completed: compile rewrite-owned first-party host/io plus app/runtime packages and owned app/media grimoires against the new frontend, trim imported `std` back to approved responsibilities, move showcase-specific helpers out where needed, and freeze std shape for pre-selfhost unless Milestone 6/runtime work proves a blocker.
6. Completed: replace the placeholder backend artifact path with a real internal backend contract, add the first narrow runtime execution lane for `hello`-class host-core routines, then finish the first runnable backend sufficient for owned `hello`-class, host-tool, window/input/canvas showcase proof, and basic audio smoke proof, with rewrite-owned host/window/input/canvas/events/time/audio/graphics/text seams implemented in Rust as real runtime commitments rather than compatibility wrappers.
7. Completed: close runtime coverage for the approved rewrite-owned `std` surface before broad grimoire growth, including the approved host-core, collections, bytes/text, config/manifest, memory, concurrency, ECS, and app-substrate slices, with acceptance driven by approved std scopes/status rather than archived corpus.
8. Completed: turn the backend/AOT path into real emitted artifacts: native `exe` / `dll` outputs plus the build/run packaging needed for real host execution, visual window/showcase smoke, and non-synthetic audio smoke, so AOT is no longer only an internal artifact format. The Milestone 6/7 findings program in `docs/reference/audits/milestone_6_7_findings_closure_matrix_v1.md` is complete; the pre-native hardening follow-through for alias-normalized artifact metadata, capability derivation semantics for non-entrypoint/library artifacts, stricter artifact structural validation, target-aware lockfile/manifest support, and exported-surface validation is complete, and native emission now lands as package-specific Windows bundles with real host execution plus emitted-binary visual/audio smoke, so later work here is post-Milestone-8 hardening or backend-retargeting rather than a missing native artifact lane.
9. Complete the rewrite-owned app/media grimoires under `grimoires/owned/*` and any other needed Arcana-owned package layers on the new toolchain after approved `std` runtime closure and runnable artifact support are in place; this step is now in progress starting with `grimoires/owned/libs/arcana-desktop`, and scaffold status alone does not satisfy it.
10. Build and native-run at least one real owned showcase app through those completed Arcana-owned grimoires on the post-Milestone-8 artifact lane, so showcase proof is no longer only direct substrate smoke through `std.*`.
11. Add first-party `arcana test` and `arcana format` to the rewrite-owned CLI before selfhost; keep `arcana check` as the correctness gate.
12. Add the smaller advisory `arcana review` layer only after the rewrite has enough real showcase/tooling corpus to justify it; keep it intentionally smaller than compiler-owned correctness checks and grounded in spec-backed or corpus-backed guidance rather than premature heuristics.
13. Validate selfhost closure against the owned Arcana compiler/tooling corpus plus explicit conformance fixtures without preserving Meadow-era package decomposition by default.
14. Keep the archived historical corpus out of the default development loop and use manual out-of-tree migration checks only where they still provide concrete value before final selfhost closure.

## Non-Goals Before Selfhost

- no language expansion
- no Git or registry dependencies
- no public bytecode compatibility promise
